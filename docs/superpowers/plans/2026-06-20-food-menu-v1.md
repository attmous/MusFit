# Food Menu V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Food tab feel like a complete Lifesum-inspired food diary with saved foods, meal sections, barcode/manual entry, and quick calorie logging.

**Architecture:** Keep the existing Room schema and local-first repository pattern. Add read models and write methods around `FoodRepository`, then drive a richer `FoodViewModel` state into a redesigned Compose screen.

**Tech Stack:** Android Kotlin, Jetpack Compose Material 3, Room, Hilt, Kotlin Flow, JUnit/Robolectric.

## Global Constraints

- Android only.
- Local-first MVP; no accounts, cloud sync, analytics, subscriptions, social features, AI coaching, or direct wearable cloud APIs.
- Food database stores reusable foods created manually or from barcode lookup.
- Food diary has Breakfast, Lunch, Dinner, and Snacks sections.
- Quick calorie logging must be available from Food without forcing full food database entry.
- Avoid exact Lifesum asset/layout copying; use a similar information architecture with original styling.
